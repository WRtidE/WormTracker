## 1. 系统概述
本系统是一个基于计算机视觉的自动化分析工具，专为处理微流控芯片中的线虫运动视频而设计。系统通过动态网格自适应、高敏背景减除、智能坐标聚类以及垂直优先追踪策略，能够并行且高精度地对32个物理通道内的线虫进行独立追踪与“越线计数”。

系统内置了抗干扰的“熔断机制”，能够有效应对设备震动、焦点漂移等突发性环境噪音，确保长时间大规模实验数据的准确性与鲁棒性。

## 2. 核心架构与创新机制

### 2.1 动态网格自适应与提取

微流控芯片在显微视野中可能会出现轻微的偏移或形变。系统并非使用硬编码的固定坐标，而是通过 `get_interpolated_grid` 模块在预热期动态计算物理通道边界。

- **边缘检测定位**：利用 Sobel 算子（X方向）提取通道壁的垂直边缘。
- **信号平滑**：通过列像素求和与一维卷积平滑，定位真实有效的物理边界。
- **等分插值**：结合 `NUM_CHANNELS` 和 `WALL_RATIO` 比例，精准推算出 32 个通道以及通道壁的绝对像素坐标，并建立掩膜以排除通道壁上的噪点。

### 2.2 目标检测与智能聚类

高速运动的线虫在视频流中极易产生残影，导致首尾断裂，被传统算法误判为两个目标。

- **高敏背景学习**：使用 `MOG2` 背景减除器，结合较低的阈值（`varThreshold=16`），确保高速残影也能被捕获。
- **非对称距离聚类（防跨通道误杀）**：
  - **极严水平限制 (`MERGE_DIST_X = 20`)**：严格限制 X 轴向上的合并，防止相邻通道的线虫被错误粘连。
  - **宽容垂直限制 (`MERGE_DIST_Y = 55`)**：在 Y 轴上允许较大跨度，将断裂的线虫头部和尾部成功合并为单一质心，输出稳定的中心坐标。

### 2.3 垂直优先追踪策略 


- 系统舍弃了单纯的欧几里得距离，将 X 与 Y 的运动容忍度解耦。
- 严格限制水平最大位移 (`MAX_DIST_X = 25`)，防止线虫 ID 跃迁到隔壁通道。
- 极度宽容垂直位移 (`MAX_DIST_Y = 300`)，以适应目标突然加速或掉落的情况。

### 2.4 熔断机制 (Panic Circuit Breaker)

针对实际实验环境中常见的物理碰撞或显微镜调焦干扰，系统设计了设备防抖防崩溃机制：

- **探针 1：网格崩溃检测**：如果当前帧计算出的通道宽度相对于基准宽度发生巨大突变（> 15%），判定为视野剧变。
- **探针 2：全图噪点检测**：如果前景掩膜面积占比超过阈值（> 1.5%），判定为光照剧变或震动。
- **熔断保护**：一旦触发，系统立即挂起追踪器与计数器，进入 `COOLDOWN_FRAMES`（30帧）的冷却期。在冷却期内，强制调高背景学习率（`learningRate=0.1`），使系统快速适应震动后的新背景，彻底杜绝全局噪点引发的计数雪崩。

## 3. 全局参数配置说明

| **参数分类**   | **参数名**              | **默认值**   | **作用与解释**                                               |
| -------------- | ----------------------- | ------------ | ------------------------------------------------------------ |
| **物理映射**   | `NUM_CHANNELS`          | 32           | 微流控芯片的实际并行通道数。                                 |
|                | `PHYSICAL_LABELS`       | [33...1]     | 对应的真实物理标签。                  |
|                | `WALL_RATIO`            | 0.0          | 物理墙壁厚度占通道的比例。                                   |
| **视野裁剪**   | `X_MARGIN`              | (0.05, 0.15) | 屏蔽画面最左侧 5% 和最右侧 15% 的无效视野。                  |
| **计数判定**   | `TRIPWIRE_RATIO`        | 0.65         | 虚拟检测线位于画面高度的 65% 处。自下而上穿过此线即触发计数。 |
|                | `CROSS_DEBOUNCE`        | 20           | 越线防抖帧数。目标越线后锁定 20 帧，防止在红线处反复徘徊导致重复计数。 |
| **形态与追踪** | `MIN_AREA` / `MAX_AREA` | 15 / 4500    | 目标的合法像素面积区间。                                     |
|                | `MAX_DIST_X`            | 25 px        | 相邻两帧允许的最大水平横跳距离。                             |
|                | `MAX_DIST_Y`            | 300 px       | 相邻两帧允许的最大垂直跃迁距离。                             |
| **熔断阈值**   | `PANIC_NOISE_RATIO`     | 0.015        | 全局突变像素占比超过 1.5% 触发熔断。                         |
|                | `GRID_MUTATION...`      | 0.15         | 网格突变超过 15% 触发熔断。                                  |
|                | `COOLDOWN_FRAMES`       | 30           | 触发熔断后的强制暂停处理帧数。                               |

## 4. 核心功能模块解析

### 函数`get_interpolated_grid(frame, ...)`

- **输入**：原始视频帧、物理通道数、裁剪比例等。
- **处理逻辑**：转灰度 -> 截取中央区域 -> 索贝尔求导 -> 垂直投影计算像素密度 -> 寻找第一个和最后一个合法波峰 -> 结合 `WALL_RATIO` 等距划分坐标。
- **输出**：包含 ROI 边界、通道宽度、墙壁宽度、各个通道矩形边界和墙壁矩形边界的字典。

### 函数`detect_worms(frame, backSub, grid_data, min_area)`

- **输入**：当前帧、MOG2 背景模型、网格数据。
- **处理逻辑**：
  1. 应用背景减除获得原始前景。
  2. 利用 `grid_data` 生成 `safe_mask`，屏蔽墙壁区域的噪点。
  3. 应用两次闭运算（Morphology Close）连接细小断点。
  4. 提取所有大于 `min_area` 的轮廓，计算初步质心集合。
  5. 执行“非对称距离合并算法”：遍历质心，若横向距离 `< MERGE_DIST_X` 且纵向距离 `< MERGE_DIST_Y`，则合并为同一目标并取平均坐标。
- **输出**：修正后的质心列表、降噪后的前景掩膜、全局噪点比例、原始前景。

### 函数`update_tracks(current_centroids, active_worms, ...)`

- **处理逻辑**：接收当前帧的检测质心，遍历已有的活动追踪器 (`active_worms`)。采用基于距离的贪心匹配，寻找满足 `MAX_DIST_X` 和 `MAX_DIST_Y` 限制且 Y 轴距离最近的轨迹进行更新。未匹配的质心分配新 ID，未更新的旧轨迹自动消亡。

### 函数`count_crossings(active_worms, counts_dict, ...)`

- **处理逻辑**：遍历活动轨迹，对比上一帧 (`p1`) 和当前帧 (`p2`) 的 Y 坐标。若满足 `p1.y >= tripwire_y` 且 `p2.y < tripwire_y`，触发越线判定。
- **精准归属**：系统计算当前发生越线时的 X 坐标，遍历并计算其与所有物理通道中心点的距离，将计数准确归属给距离最小的通道，避免边界模糊区域的误判。

### 函数`process_video(video_path)`

- **核心主循环**：
  1. **预热期 (前 30 帧)**：不进行追踪，仅静默学习背景模型，并在第 30 帧锁定“基准网格”。
  2. **探针检查**：每帧验证网格形态和画面噪点，判断是否需要进入熔断状态。
  3. **正常流水线**：若无异常，依次执行网格自适应 -> 目标检测 -> 轨迹更新 -> 越线判定 -> UI渲染。
  4. **数据落盘**：处理结束后，汇总并格式化输出数据，导出为同名的 CSV 报告。

## 5. 用户交互与数据输出

### 实时交互控制

在系统运行时，聚焦于视频显示窗口，可使用以下快捷键：

- **`Space` (空格键)**：暂停/继续视频处理。
- **`S` 键**：无损保存当前帧的带标注画面到工作目录 (格式: `screenshot_XXXX.png`)。
- **`Q` 键**：提前终止处理并保存当前统计数据。

### 数据输出 (`.csv`)

处理完成后，系统会自动在视频所在目录生成同名附加 `_counts.csv` 的数据文件。

文件结构包含两列：`通道编号` 与 `线虫数量`。底部包含本次实验的 `总计` 数值，可直接导入 Excel、Origin 或 GraphPad Prism 进行后续生物统计学分析。

## 6. 环境依赖要求

要运行本系统，需要配置基础的 Python 机器视觉环境：

- **Python版本**：`Python 3.7+`
- **核心依赖包**：
  - `opencv-python` (提供 cv2 视频读取、形态学操作、背景建模及 UI 绘制)
  - `numpy` (提供高速矩阵运算与一维信号平滑处理)
- **内置模块**：`csv`, `collections.deque` (Python自带，无需安装)
